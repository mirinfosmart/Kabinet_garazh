# Шаблон HTTP-сервісу 1С — `gsk`

> Код-приклад модуля HTTP-сервісу для 1С:Підприємство 8.3 (керовані форми).
> Програміст 1С підставляє реальні запити до регістрів/довідників замість заглушок.
>
> Працює у парі з контрактом **`API_1C_CONTRACT.md`**.

---

## 1. Налаштування HTTP-сервісу в конфігураторі

1. **Дерево конфігурації → HTTP-сервіси → Додати.**
   - Ім'я: `gsk`
   - Кореневий URL: `gsk`

2. **Додати шаблони URL** (для кожного — свій метод):

   | Шаблон URL | Метод HTTP | Ім'я обробника |
   |------------|-----------|----------------|
   | `/owner` | GET | `ownerGET` |
   | `/balance` | GET | `balanceGET` |
   | `/charges` | GET | `chargesGET` |
   | `/payments` | GET | `paymentsGET` |
   | `/electricity` | GET | `electricityGET` |
   | `/notices` | GET | `noticesGET` |
   | `/electricity/submit` | POST | `electricitySubmitPOST` |

3. **Публікація:** Адміністрування → Публікація на веб-сервері. Увімкнути «Публікувати HTTP-сервіси»,
   відмітити сервіс `gsk`.

4. **Basic Auth:** на рівні публікації (web.config / .htaccess веб-сервера) увімкнути базову
   автентифікацію; 1С перевіряє користувача автоматично. Заведіть **окремого технічного користувача**
   1С з мінімальними правами (тільки читання потрібних довідників/регістрів і запис показань).

## 2. Загальний модуль `ГСК_API_Сервіс` (допоміжні функції)

Створіть загальний модуль (галочки «Сервер», «Виклик сервера») і додайте:

```bsl
// Формує HTTPСервисОтвет з JSON-тілом та потрібними заголовками.
Функция СформироватьJSONОтвет(Знач Данные, Знач КодСостояния = 200) Экспорт

    Ответ = Новый HTTPСервисОтвет(КодСостояния);

    ЗаписьJSON = Новый ЗаписьJSON;
    Поток = Новый ПотокВПамяти;
    ЗаписьJSON.ОткрытьПоток(Поток, "UTF-8");
    // ЭкранированиеСимволовUTF8 — кирилиця піде як є, без \uXXXX
    НастройкиСериализации = Новый НастройкиСериализацииJSON;
    НастройкиСериализации.ИспользоватьЭкранированиеURL = Ложь;
    ЗаписатьJSON(ЗаписьJSON, Данные);
    ЗаписьJSON.Закрыть();

    Ответ.УстановитьТелоИзДвоичныхДанных(Поток.ЗакрытьИПолучитьДвоичныеДанные());
    Ответ.Заголовки.Вставить("Content-Type", "application/json; charset=utf-8");

    Возврат Ответ;

КонецФункции

// Формує стандартну відповідь-помилку за контрактом (§2).
Функция ОшибкаОтвет(Знач Код, Знач Сообщение, Знач HTTPКод) Экспорт

    Данные = Новый Структура;
    Данные.Вставить("error",   Истина);
    Данные.Вставить("code",    Код);
    Данные.Вставить("message", Сообщение);

    Возврат СформироватьJSONОтвет(Данные, HTTPКод);

КонецФункции

// Читає та валідує обов'язковий параметр garageNumber із запиту.
// Повертає Истина якщо ок; інакше заповнює ВыходнойОтвет помилкою.
Функция ПрочитатьНомерГаража(Запрос, НомерГаража, ВыходнойОтвет) Экспорт

    НомерГаража = Запрос.ПараметрыЗапроса.Получить("garageNumber");

    Если НЕ ЗначениеЗаполнено(НомерГаража) Тогда
        ВыходнойОтвет = ОшибкаОтвет("BAD_REQUEST",
            "Не вказано номер гаража (garageNumber)", 400);
        Возврат Ложь;
    КонецЕсли;

    Возврат Истина;

КонецФункции

// Читає JSON-тіло POST-запиту у структуру.
Функция ПрочитатьТелоJSON(Запрос) Экспорт

    Текст = Запрос.ПолучитьТелоКакСтроку("UTF-8");

    ЧтениеJSON = Новый ЧтениеJSON;
    ЧтениеJSON.УстановитьСтроку(Текст);
    Данные = ПрочитатьJSON(ЧтениеJSON, Истина); // Истина = у Соответствие/Структуру
    ЧтениеJSON.Закрыть();

    Возврат Данные;

КонецФункции
```

> **Дати:** для формату `YYYY-MM-DD` використовуйте
> `Формат(ДатаЗначення, "ДФ=yyyy-MM-dd")`.
> **id як рядок:** `Формат(Ссылка.Код, "ЧГ=0")` або `XMLСтрока(Ссылка.УникальныйИдентификатор())`.

## 4. Швидка перевірка (без додатку)

Перевірте кожен метод через браузер або `curl` ще до підключення додатку:

```bash
# GET (логін:пароль через -u)
curl -u gsk_user:пароль \
  "http://192.168.1.100:8080/gsk_base/hs/gsk/owner?garageNumber=А-47"

# POST
curl -u gsk_user:пароль -X POST \
  -H "Content-Type: application/json" \
  -d '{"garageNumber":"А-47","reading":251.3,"date":"2026-06-25"}' \
  "http://192.168.1.100:8080/gsk_base/hs/gsk/electricity/submit"
```

Якщо `curl` повертає валідний JSON за контрактом — додаток підключиться без змін
(достатньо в `.env` поставити `EXPO_PUBLIC_USE_MOCK=false` і вказати `EXPO_PUBLIC_1C_BASE_URL`).

## 5. Часті граблі

| Симптом | Причина / рішення |
|---------|-------------------|
| `401` навіть з вірним паролем | Basic Auth не увімкнено в публікації, або користувач без права на сервіс |
| Кирилиця як `Ð...` | Не виставлено `charset=utf-8` у Content-Type |
| Додаток: «Network request failed» | Сервер недоступний з телефону (інша підмережа / firewall / потрібен port forwarding) |
| Дати показуються як `[object]` | Дата віддана як тип Дата 1С, а не рядок `YYYY-MM-DD` |
| `id` не співпадає / дублі ключів | `id` віддано числом; має бути рядок |
| Порожній екран замість «немає даних» | Повернуто `null` замість `[]` |
```

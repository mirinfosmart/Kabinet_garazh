# Контракт API — HTTP-сервіс 1С для додатку «Мій Гараж» (версія v2)

> Документ для програміста 1С. Точні формати запитів і відповідей HTTP-сервісу `gsk`,
> який споживає мобільний додаток (React Native / Expo).
>
> **v2 vs v1.** Авторизація тепер за **телефоном + OTP** із видачею **Bearer-токена**.
> Номер гаража **більше не передається** у запитах даних — сервер визначає його з токена.
> Це усуває ваду v1, де будь-хто міг підставити чужий `garageNumber`.
>
> Реальні об'єкти метаданих BAF, до яких прив'язані методи, описано в `../ПРОЕКТ_Особистий_кабінет_ГСК.md`.
> Код-приклади обробників — у `1C/ГСК_API_Сервіс.bsl` та `1C/HTTPСервіс_gsk_Модуль.bsl`.

---

## 1. Загальні правила

| Параметр | Значення |
|----------|----------|
| Базовий URL | `https://<сервер>/<база>/hs/gsk` |
| Кодування | UTF-8 |
| Формат тіла | `application/json; charset=utf-8` |
| Транспорт | HTTPS обов'язково. Опційно Basic Auth тех. користувача публікації |
| Авторизація власника | `Authorization: Bearer <token>` (для всіх методів, крім `/auth/*`) |
| Таймаут клієнта | 10 секунд |

Дати — рядок `YYYY-MM-DD`. Суми — `number`. `id` — **рядок**. Порожній список — `[]`, не `null`.

---

## 2. Формат помилок (єдиний)

```json
{ "error": true, "code": "GARAGE_NOT_FOUND", "message": "Текст українською" }
```

| HTTP | `code` | Коли |
|------|--------|------|
| 400 | `BAD_REQUEST` | Немає обов'язкового поля |
| 401 | `UNAUTHORIZED` / `TOKEN_EXPIRED` | Немає/недійсний/прострочений токен |
| 403 | `OTP_INVALID` / `OTP_LOCKED` | Невірний код / перевищено спроби |
| 404 | `GARAGE_NOT_FOUND` | Гараж не знайдено |
| 422 | `VALIDATION_ERROR` | Невалідні дані (показник < попереднього) |
| 500 | `INTERNAL_ERROR` | Помилка 1С |

---

## 3. Авторизація

### 3.1. `POST /auth/request-otp`

**Тіло:** `{ "phone": "+380671234567" }`

**Відповідь `200`:**
```json
{ "sent": true, "maskedPhone": "+380•• ••• 45 67", "expiresIn": 300 }
```
> З міркувань безпеки сервіс відповідає `sent:true` навіть якщо телефон не знайдено (захист від перебору). Код надсилається лише за наявності гаража з таким `НомерТелефона`.

### 3.2. `POST /auth/verify-otp`

**Тіло:** `{ "phone": "+380671234567", "code": "1234" }`

**Відповідь `200`:**
```json
{ "token": "<64-символьний hex>", "garageNumber": "А-47", "fullName": "Ковальчук Олег Михайлович" }
```
Помилки: `403 OTP_INVALID`, `403 OTP_LOCKED`, `404 GARAGE_NOT_FOUND`.

Токен діє 30 днів (зберігається в `РегістрВідомостей.ГСК_СессииКабинета`). Клієнт додає його в заголовок усіх подальших запитів.

---

## 4. Методи даних (потрібен `Authorization: Bearer <token>`)

> Гараж у цих запитах **не передається** — береться з токена на сервері.

### 4.1. `GET /me`
```json
{ "fullName": "Ковальчук Олег Михайлович", "garageNumber": "А-47",
  "cooperative": "ГСК «Автомобіліст»", "phone": "+380671234567", "memberSince": "2019" }
```

### 4.2. `GET /balance`
```json
{ "balance": -1240, "debt": 1240, "debtMonths": 2, "penalty": 120,
  "lastPaymentDate": "2026-05-15", "nextDueDate": "2026-07-01" }
```
`balance < 0` = борг, `> 0` = переплата. Джерело: `РегістрНакопичення.РасчетыПоВладельцамГаражей` (вид Остатки).

### 4.3. `GET /charges` → масив
```json
[ { "id": "1", "name": "Членський внесок", "amount": 620,
    "dueDate": "2026-06-01", "status": "overdue", "period": "Травень 2026" } ]
```
`status ∈ {paid, overdue, pending}`.

### 4.4. `GET /payments?limit=20` → масив
```json
[ { "id": "1", "date": "2026-05-15", "amount": 620, "type": "Членський внесок",
    "period": "Квітень 2026", "status": "paid" } ]
```
Джерело: `Документ.ПриходныйКассовыйОрдерГаражи` (відбір по `Контрагент = ВладелецГаража`).

### 4.5. `GET /electricity`
```json
{ "currentReading": 245.8, "previousReading": 198.4, "consumed": 47.4,
  "rate": 4.32, "amount": 204.77, "lastReadDate": "2026-06-01" }
```
Джерело: остання ТЧ `Документ.ВводПоказанийПриборовУчета`.

### 4.6. `POST /electricity/submit`

**Тіло:** `{ "reading": 251.3, "date": "2026-06-25" }` (без `garageNumber`!)

**Відповідь `200`:**
```json
{ "success": true, "accepted": 251.3, "consumed": 52.9, "message": "Показники прийнято" }
```
**`422`** якщо показник менший за попередній. Запис іде у проміжний `РегістрВідомостей.ПоказанияПриборовУчетаИзМобильногоКабинета`; бухгалтер формує `Документ.ВводПоказанийПриборовУчета` з `Источник = МобильноеПриложение`.

### 4.7. `GET /notices` → масив
```json
[ { "id": "1", "date": "2026-06-18", "title": "Збори кооперативу",
    "text": "...", "type": "meeting", "isRead": false } ]
```
`type ∈ {meeting, finance, info}`. ⚠️ У гаражній підсистемі BAF поки немає об'єкта «Оголошення» — метод повертає `[]`, доки об'єкт не визначено в Конфігураторі (див. проєкт-документ).

---

## 5. Довідник enum-значень

**`status`** (charges/payments): `paid` (Оплачено, зелений) · `overdue` (Прострочено, червоний) · `pending` (До оплати, жовтий).

**`type`** (notices): `meeting` 📢 · `finance` 💰 · `info` ℹ️.

---

## 6. Чек-лист для програміста 1С

- [ ] Створено HTTP-сервіс `gsk` (RootURL `gsk`) з шаблонами URL §4 + `/auth/*`.
- [ ] Додано регістри `ГСК_КодыПодтверждения`, `ГСК_СессииКабинета`, `ПоказанияПриборовУчетаИзМобильногоКабинета`.
- [ ] Перенесено код із `1C/ГСК_API_Сервіс.bsl` та `1C/HTTPСервіс_gsk_Модуль.bsl`.
- [ ] Підключено канал доставки OTP (`ДоставитьКодПодтверждения`).
- [ ] Звірено напрям руху `РасчетыПоВладельцамГаражей` (Приход = борг) з обліковцем.
- [ ] Уточнено джерела `cooperative`, `rate`, `penalty`, `nextDueDate` (позначені TODO у коді).
- [ ] Опубліковано на HTTPS-веб-сервері, перевірено `curl`.

```bash
# 1) запит коду
curl -X POST -H "Content-Type: application/json" \
  -d '{"phone":"+380671234567"}' \
  "https://192.168.1.100/gsk_base/hs/gsk/auth/request-otp"

# 2) підтвердження → отримати token
curl -X POST -H "Content-Type: application/json" \
  -d '{"phone":"+380671234567","code":"1234"}' \
  "https://192.168.1.100/gsk_base/hs/gsk/auth/verify-otp"

# 3) дані з токеном
curl -H "Authorization: Bearer <token>" \
  "https://192.168.1.100/gsk_base/hs/gsk/balance"
```
